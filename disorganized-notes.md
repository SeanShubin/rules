- scope
    - human
        - decisions
        - verifying what is true
        - automated tests
        - test orchestrator pattern
        - identifying research needs
        - ownership of commit
            - do not push a commit until
                - you understand the rationale
                - you have verified the premises behind the decision
                - you agree with the conclusion
    - llm
        - previously done by humans
            - prototyping
            - automated test infrastructure (stubs, fakes)
            - compiling and organizing research data
            - mechanical code transformations once a decision has been made
        - previously not done at all
            - special purpose or short-term utilities
            - reorganizing documentation
            - vetting documentation for inconsistencies
    - tooling
        - general purpose, long-term, utilities
        - quality metrics
        - automation

- Sean's process rules
    - every ai task on separate commit
    - prioritize putting context into self documenting, maintainable code
    - create documentation for rules ai should apply (load on demand)
    - create documentation for context ai should take into account for certain tasks (load on demand)
    - have an editor open for my ai prompts, to break them up in to smaller chunks, and work on them while the ai is doing something
    - press ai on anything I don't understand, until I understand it
    - press ai on anything I have not verified is true, until I have verified it is true
    - for things I know how to solve, ask the ai how it would solve it before telling it my solution
    - make all decisions
    - fully review entire git diff before committing
    - monitor what ai is doing as it is doing it, and update prompts document with follow up questions while I wait
    - On production code, I allow the ai to make a mess of things between commits, but only commit code I would have written myself
    - For temporary tooling, I rely on it actually being useful to me without looking at how the ai did it
    - For prototyping, I don't apply quality standards
    - When you don't like the way something works, ask the ai to recommend alternatives
